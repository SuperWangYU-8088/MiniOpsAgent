# MiniOpsAgent 使用说明

本文档说明 MiniOpsAgent 的日常使用方式、配置项、命令、状态目录和常见问题。

## 1. 命令总览

```bash
miniopsagent [flags]
miniopsagent version
miniopsagent doctor
miniopsagent index [path]
miniopsagent search <query>
miniopsagent graph
miniopsagent serve --port 8080
miniopsagent snapshot list
miniopsagent snapshot restore <id>
```

开发阶段可以直接使用：

```bash
go run ./cmd/miniopsagent doctor
```

## 2. 根命令参数

| 参数 | 说明 |
| --- | --- |
| `--once` | 执行一次 prompt 后退出 |
| `--plain` | 使用普通 stdin/stdout REPL |
| `--cwd` | 切换工作区目录 |
| `--mode` | 指定运行模式：`react`、`plan`、`team` |

示例：

```bash
go run ./cmd/miniopsagent --cwd /opt/app --once "检查这个项目有哪些部署脚本"
```

## 3. 运行模式

### react

默认模式。模型可以在最多 10 轮内调用工具、读取结果并回答。

```bash
go run ./cmd/miniopsagent --once "找出项目里所有 Dockerfile"
```

### plan

先生成计划，再按计划执行。

```bash
go run ./cmd/miniopsagent --mode plan --once "整理项目部署步骤"
```

也可以在交互中使用：

```text
/plan 整理项目部署步骤
```

### team

用 Planner、Worker、Reviewer 三个角色串行分析，适合学习多角色协作形态。

```bash
go run ./cmd/miniopsagent --mode team --once "从运维角度评审这个项目"
```

## 4. 配置项

统一环境变量：

| 变量 | 说明 |
| --- | --- |
| `MINIOPS_PROVIDER` | 默认模型供应商 |
| `MINIOPS_API_KEY` | 当前供应商 API key |
| `MINIOPS_BASE_URL` | 当前供应商 OpenAI-compatible base URL |
| `MINIOPS_MODEL` | 当前供应商模型名 |
| `MINIOPS_RUNTIME_API_KEY` | Runtime API 访问密钥 |

provider 专属变量：

```text
DEEPSEEK_API_KEY
OPENAI_API_KEY
GLM_API_KEY
KIMI_API_KEY
STEP_API_KEY
AGNES_API_KEY
XFYUN_MAAS_API_KEY
```

provider 专属模型和地址：

```text
DEEPSEEK_MODEL
DEEPSEEK_BASE_URL
OPENAI_MODEL
OPENAI_BASE_URL
```

## 5. MCP 配置

用户级：

```text
~/.miniopsagent/mcp.json
```

项目级：

```text
.miniopsagent/mcp.json
```

示例：

```json
{
  "mcpServers": {
    "example": {
      "command": "node",
      "args": ["server.js"],
      "env": {
        "PROJECT_DIR": "${PROJECT_DIR}"
      }
    }
  }
}
```

远端 HTTP MCP 示例：

```json
{
  "mcpServers": {
    "remote_search": {
      "url": "https://example.com/mcp",
      "headers": {
        "Authorization": "Bearer ${REMOTE_MCP_TOKEN}"
      }
    }
  }
}
```

注册后的工具名称格式：

```text
mcp__<server-name>__<tool-name>
```

## 6. Skill 配置

用户级 Skill：

```text
~/.miniopsagent/skills/<skill-name>/SKILL.md
```

项目级 Skill：

```text
.miniopsagent/skills/<skill-name>/SKILL.md
```

示例：

```markdown
---
name: linux-triage
description: Linux 故障排查流程，包括磁盘、内存、进程、端口和日志检查。
version: "1.0.0"
author: MiniOpsAgent
tags: [linux, triage, ops]
---

# Linux Triage

优先读取系统信息，再检查磁盘、内存、端口、进程和最近日志。
```

模型会先看到 Skill 索引，需要完整内容时再调用 `load_skill`。

## 7. 快照和恢复

每次 Agent turn 开始前会创建 `pre-turn` 快照，结束后会创建 `post-turn` 快照。

列出快照：

```bash
go run ./cmd/miniopsagent snapshot list
```

恢复：

```bash
go run ./cmd/miniopsagent snapshot restore <id>
```

注意：当前快照恢复是基础 copy 实现，适合学习和小项目，不等同于 Git 完整回滚。

## 8. 审计日志

危险工具和 MCP 工具调用会记录到：

```text
~/.miniopsagent/audit/YYYY-MM-DD.jsonl
```

记录字段：

```json
{
  "time": "2026-07-22T12:00:00Z",
  "tool": "execute_command",
  "outcome": "allow",
  "approver": "policy",
  "args_hash": "hash",
  "duration": "1.2s"
}
```

审计里只保存参数 hash，不保存完整命令或文件内容，避免日志泄露敏感信息。

## 9. 常见问题

### go 命令不存在

安装 Go，并确保 `go` 在 PATH 中。

### doctor 显示 API key missing

检查：

```bash
echo $MINIOPS_API_KEY
echo $DEEPSEEK_API_KEY
```

PowerShell：

```powershell
echo $env:MINIOPS_API_KEY
```

### web_search 不稳定

DuckDuckGo fallback 可能被网络限制。建议配置：

```bash
export SERPAPI_API_KEY=...
```

或者：

```bash
export SEARXNG_BASE_URL=http://你的-searxng:8080
```

### execute_command 在 Windows 不执行 Bash 语法

Windows 下使用 PowerShell 执行命令。Linux/macOS 下使用 `sh -lc`。因此跨平台命令建议使用简单命令，避免把 Bash 专属语法直接放到 Windows 环境中。
