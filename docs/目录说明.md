# 目录说明

这份文档用来解释 MiniOpsAgent 仓库里每个主要目录的职责，方便你从运维学习、代码改造和后续维护三个角度去看项目。

## 根目录

| 路径 | 说明 |
| --- | --- |
| `README.md` | 项目总入口，包含定位、快速开始、配置方式、目录结构和安全说明。 |
| `go.mod` | Go 模块定义，决定项目导入路径和依赖版本。 |
| `.gitignore` | Git 忽略规则，避免把本地状态、构建产物和运行时文件提交到仓库。 |
| `docs/` | 项目文档目录，放教程、说明、设计和实现细节。 |

## 核心目录

| 路径 | 说明 |
| --- | --- |
| `cmd/miniopsagent/` | CLI 入口目录，负责解析命令行参数并启动主程序。 |
| `internal/agent/` | Agent 核心调度层，负责对话循环、模式切换、记忆注入和工具调用编排。 |
| `internal/config/` | 配置加载层，负责读取 `.env`、配置文件和环境变量，并合并 provider 默认值。 |
| `internal/llm/` | 大模型客户端封装，处理 OpenAI-compatible 接口、流式输出和消息格式转换。 |
| `internal/tools/` | 工具注册与实现层，包含文件、命令、Web、MCP、RAG、Skill、快照等能力。 |
| `internal/rag/` | 本地代码索引与检索层，用于构建轻量 RAG 索引和代码搜索。 |
| `internal/skill/` | Skill 扫描与加载层，负责读取 `SKILL.md` 并按需注入能力说明。 |
| `internal/snapshot/` | 快照层，负责 turn 前后状态保存与恢复。 |
| `internal/runtime/` | 本地 Runtime API 服务层，对外提供 thread、turn 和 events 接口。 |
| `internal/tui/` | 终端界面层，负责 Bubble Tea 界面的布局、输入、输出和状态展示。 |

## 文档目录

| 路径 | 说明 |
| --- | --- |
| `docs/入门教程.md` | 面向新手的上手路线，讲如何启动、配置和第一次使用。 |

## 运行时目录

运行后，用户态状态默认会放在 `~/.miniopsagent/`，例如配置、MCP、Skill、RAG 索引、快照和审计日志。  
如果项目级目录里存在 `.miniopsagent/`，它通常用于存放项目级 MCP 和 Skill 配置，便于把能力跟仓库一起管理。

## 你可以怎么用这份目录说明

1. 先看 `cmd/miniopsagent/` 和 `internal/agent/`，了解程序入口和主流程。
2. 再看 `internal/tools/` 和 `internal/runtime/`，了解它到底能做什么。
3. 最后看 `docs/架构设计.md` 和 `docs/实现说明.md`，把功能和代码对应起来。
