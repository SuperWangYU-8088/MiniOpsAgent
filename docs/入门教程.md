# MiniOpsAgent 入门教程

本文档面向第一次接触 MiniOpsAgent 的学习者，目标是带你从“项目能跑起来”走到“能看懂一次 Agent 调用链路”。

## 1. 项目是什么

MiniOpsAgent 是一个运行在终端里的轻量级运维 Agent。它把大模型和本地工具连接起来，让模型可以在你的工作区里完成以下动作：

- 阅读文件和目录。
- 搜索代码。
- 执行受策略约束的命令。
- 联网搜索和抓取网页。
- 查询本地代码索引。
- 加载 Skill。
- 调用 MCP 工具。
- 创建和恢复工作区快照。
- 通过 HTTP Runtime API 接收外部请求。

你可以把它理解为一个“小型可学习的运维 Agent 原型”，不是完整生产平台。

## 2. 准备环境

推荐环境：

```text
Go 1.26+
PowerShell / Bash
rg 可选但推荐
可访问 OpenAI-compatible 模型服务
```

检查 Go：

```bash
go version
```

如果输出版本号，说明 Go 已经可用。如果命令不存在，需要先安装 Go 并把 `go` 加到 PATH。

## 3. 配置模型

最简单的方式是使用统一环境变量：

```bash
export MINIOPS_PROVIDER=deepseek
export MINIOPS_API_KEY=你的密钥
export MINIOPS_MODEL=deepseek-chat
```

PowerShell 示例：

```powershell
$env:MINIOPS_PROVIDER = "deepseek"
$env:MINIOPS_API_KEY = "你的密钥"
$env:MINIOPS_MODEL = "deepseek-chat"
```

也可以使用 provider 专属变量：

```bash
export DEEPSEEK_API_KEY=...
export OPENAI_API_KEY=...
```

配置文件路径：

```text
~/.miniopsagent/config.json
```

示例：

```json
{
  "default_provider": "deepseek",
  "providers": {
    "deepseek": {
      "api_key": "替换成真实密钥",
      "base_url": "https://api.deepseek.com/v1",
      "model": "deepseek-chat",
      "max_context": 128000
    }
  }
}
```

## 4. 运行 doctor

进入项目根目录：

```bash
cd MiniOpsAgent
go run ./cmd/miniopsagent doctor
```

你会看到：

```text
MiniOpsAgent: 0.1.0-go
CWD: 当前工作区
State dir: ~/.miniopsagent
Provider: deepseek
Model: deepseek-chat
Base URL: present
API key: present
...
```

如果 API key 是 `missing`，优先检查 `MINIOPS_API_KEY` 或 provider 专属变量是否设置。

## 5. 单次调用

单次调用适合脚本化验证：

```bash
go run ./cmd/miniopsagent --once "介绍一下这个项目的目录结构"
```

Plan 模式：

```bash
go run ./cmd/miniopsagent --mode plan --once "分析这个项目适合怎么学习"
```

Team 模式：

```bash
go run ./cmd/miniopsagent --mode team --once "从运维角度评审这个项目"
```

## 6. 纯文本 REPL

```bash
go run ./cmd/miniopsagent --plain
```

输入问题后回车，输入 `/exit` 退出。

常用交互命令：

```text
/help
/plan <task>
/team <task>
/exit
```

## 7. 全屏 TUI

```bash
go run ./cmd/miniopsagent
```

TUI 会显示：

- 当前模型。
- MCP 准备状态。
- Skill 数量。
- 上下文估算。
- 工具调用和工具结果。
- 最终回答。

如果终端太小，建议调大窗口。

## 8. 建立代码索引

MiniOpsAgent 自带轻量代码索引：

```bash
go run ./cmd/miniopsagent index .
```

搜索：

```bash
go run ./cmd/miniopsagent search "RunWithObserver"
```

查看关系图：

```bash
go run ./cmd/miniopsagent graph
```

索引文件保存在：

```text
~/.miniopsagent/rag/code-index.json
```

## 9. Runtime API 体验

设置 API key：

```bash
export MINIOPS_RUNTIME_API_KEY=dev-secret
```

启动服务：

```bash
go run ./cmd/miniopsagent serve --port 8080
```

创建线程：

```bash
curl -X POST http://127.0.0.1:8080/v1/threads \
  -H "Authorization: Bearer dev-secret"
```

创建任务：

```bash
curl -X POST http://127.0.0.1:8080/v1/threads/<thread-id>/turns \
  -H "Authorization: Bearer dev-secret" \
  -H "Content-Type: application/json" \
  -d '{"input":"列出项目核心模块","mode":"react"}'
```

查看事件：

```bash
curl http://127.0.0.1:8080/v1/threads/<thread-id>/events \
  -H "Authorization: Bearer dev-secret"
```

## 10. 学习顺序

推荐按这个顺序读代码：

1. `cmd/miniopsagent/main.go`：命令入口和 bootstrap。
2. `internal/config/config.go`：配置加载。
3. `internal/llm/client.go`：模型请求和流式解析。
4. `internal/agent/agent.go`：Agent 主循环。
5. `internal/tools/registry.go`：工具注册和执行。
6. `internal/tools/policy.go`：路径和命令安全策略。
7. `internal/rag/index.go`：本地代码索引。
8. `internal/runtime/server.go`：本地 HTTP API。
9. `internal/tui/tui.go`：终端 UI。
